## 컴포넌트 설계 원칙

### 카드 컴포넌트

구조:

```
┌─────────────────────────┐
│ [이모지] [카테고리태그] │ ← 상단 메타 정보
│                         │
│ 메인 제목               │ ← 쉬운 말로 작성
│ 서브 설명 (선택)        │
│                         │
│ [난이도] [예상시간]  →  │ ← 하단 정보
└─────────────────────────┘
```

원칙:

- 카드 내 텍스트는 최대 2줄 초과 시 말줄임(...) 처리
- 카드 클릭 영역은 카드 전체
- 카드 간 간격 최소 12px 이상

### 난이도 태그

🌱 씨앗 → bg: #E8F5EE / text: #4CAF82
🌿 새싹 → bg: #FFF8E1 / text: #F9A825
🌳 나무 → bg: #E3F2FD / text: #1976D2

### 버튼

Primary : bg #4CAF82 / text #FFFFFF / radius 12px
Secondary : bg #F5F5F5 / text #1A1A1A / radius 12px
Danger : bg #FF6B6B / text #FFFFFF / radius 12px
크기 : height 최소 48px (모바일 터치 영역 확보)

### 툴팁 (금융 용어 설명)

- 어려운 금융 용어 첫 등장 시 말풍선 툴팁 필수 제공
- 툴팁 내용은 초등학생도 이해할 수 있는 수준으로 작성
- 툴팁 최대 2줄 이내
  예) "전세보증금" 탭 시
  → "집주인에게 맡기는 큰 보증금이에요.
  계약이 끝나면 돌려받을 수 있어요."

## 페이지별 레이아웃 원칙

### 홈 화면

섹션 구성 순서 (고정):

1. 오늘의 한 줄 금융 상식 카드
2. 상황별 맞춤 탭 (새내기 / 취준생 / 신입사원)
3. 오늘의 퀴즈 CTA
4. 카테고리 탐색 (🏠 부동산 / 📋 세금 / 💰 재테크)
5. 인기 콘텐츠 / 신규 콘텐츠

### 콘텐츠 상세 페이지

섹션 구성 순서 (고정):

1. 30초 요약 카드 (핵심 3줄)
2. 쉬운 말 풀이 본문
3. 나에게 필요한 이유
4. 핵심 체크리스트
5. 관련 퀴즈 바로 풀기 CTA

### 퀴즈 페이지

진행 순서 (고정):

1. 배경 지식 30초 카드
2. 문제 + 선택지
3. 정답 해설 (쉬운 말)
4. 오늘 획득한 지식 요약

### 공통 레이아웃

- 모바일 최대 너비 : 390px
- 데스크탑 최대 너비 : 1200px
- 페이지 좌우 패딩 : 모바일 20px / 데스크탑 40px
- 섹션 간 간격 : 32px

## UX 원칙

### 로딩 상태

- 데이터 로딩 중에는 반드시 스켈레톤 UI를 표시한다
- 스피너 단독 사용 금지 (레이아웃 깨짐 방지)
- 로딩 시간 3초 초과 시 안내 메시지 표시

### 빈 상태 (Empty State)

- SSAC 마스코트와 함께 친근한 안내 메시지 표시
  예) 퀴즈 기록 없음
  → 🌱 "아직 푼 퀴즈가 없어요. 첫 퀴즈에 도전해볼까요?"

### 에러 상태

- 기술적 오류 메시지 노출 금지
- 사용자가 다음에 할 수 있는 행동을 함께 안내
  예) 네트워크 오류
  ❌ "NetworkError: Failed to fetch"
  ✅ "연결이 불안정해요. 잠시 후 다시 시도해주세요."

### 인터랙션

- 버튼 탭 시 즉각적인 피드백 (opacity 또는 scale 변화)
- 페이지 전환 시 fade-in 트랜지션 (300ms ease)
- 중요한 액션(탈퇴, 삭제 등)은 반드시 확인 다이얼로그 제공

### 접근성

- 모든 이미지에 alt 텍스트 필수
- 터치 영역 최소 48x48px 확보
- 키보드 탐색 가능하도록 focus 스타일 명시
- prefers-reduced-motion 미디어 쿼리 대응

### 콘텐츠 난이도 표시 원칙

- 모든 콘텐츠에 난이도 태그 필수 표시
- 난이도는 🌱 씨앗 / 🌿 새싹 / 🌳 나무 3단계로만 구분
- 예상 소요 시간 함께 표시 (예: "3분이면 충분해요")
