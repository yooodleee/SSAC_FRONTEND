# SSAC 디자인 시스템

> FE 에이전트가 UI/UX 작업 시 반드시 이 파일을 먼저 읽고 설계 지침을 준수한다.
> DESIGN.md에 정의되지 않은 디자인 결정은 임의로 하지 않고 사용자에게 확인 후 DESIGN.md에 추가한다.

---

## 브랜드 아이덴티티

### 서비스 핵심 가치

- 타겟 : 수능 끝난 고3 / 사회초년생
- 목표 : 금융 문맹 탈출의 첫 걸음
- 톤앤매너: "처음이어도 괜찮아" — 친근하고 쉽게
- 감성 : Notion 미니멀 + Apple 부드러움

### 마스코트

- 이름 : 싹 (SSAC)
- 활용 : 로딩 / 빈 상태 / 레벨업 / 온보딩 등
- 원칙 : 사용자가 어렵거나 막막할 때 등장하여 응원

### 언어 원칙

- 금융 전문 용어는 반드시 쉬운 말로 풀어 설명한다
- 문장은 짧고 친근하게 작성한다
- 어려운 용어 첫 등장 시 툴팁으로 즉시 설명한다
  예)
  ❌ "임대차 계약 시 확정일자를 취득하여야 합니다"
  ✅ "계약서 도장 찍은 날짜를 꼭 확인받으세요"

--color-primary-dark : #388E60 /_ 호버, 눌림 상태 _/

- 다크모드 대응은 CSS 변수 기반으로 처리

## 타이포그래피

### 폰트

기본 폰트 : Pretendard (한글 / 영문 모두 적용)
fallback : -apple-system, BlinkMacSystemFont, sans-serif

### 스케일

--font-size-title-lg : 28px / Bold /_ 메인 타이틀 _/
--font-size-title-md : 22px / Bold /_ 페이지 제목 _/
--font-size-title-sm : 18px / SemiBold /_ 섹션 제목 _/
--font-size-body-lg : 16px / Medium /_ 카드 제목 _/
--font-size-body-md : 15px / Regular /_ 본문 _/
--font-size-body-sm : 13px / Regular /_ 캡션 / 태그 _/

### 행간

제목 : line-height 1.3
본문 : line-height 1.6

### 사용 원칙

- 한 화면에 폰트 크기는 최대 4가지 이하로 제한
- 본문 최소 폰트 크기 13px 이하 사용 금지
- Bold는 제목 / 강조에만 사용, 본문 남용 금지

## 컴포넌트 설계 원칙

### 카드 컴포넌트

구조:
┌─────────────────────────┐
│ [이모지] [카테고리태그] │ ← 상단 메타 정보
│ │
│ 메인 제목 │ ← 쉬운 말로 작성
│ 서브 설명 (선택) │
│ │
│ [난이도] [예상시간] → │ ← 하단 정보
└─────────────────────────┘

원칙:

- 카드 내 텍스트는 최대 2줄 초과 시 말줄임(...) 처리
- 카드 클릭 영역은 카드 전체
- 카드 간 간격 최소 12px 이상

### 난이도 태그

🌱 씨앗 → bg: #E8F5EE / text: #4CAF82
🌿 새싹 → bg: #FFF8E1 / text: #F9A825
🌳 나무 → bg: #E3F2FD / text: #1976D2

### 버튼

Primary : bg #4CAF82 / text #FFFFFF / radius 12px
Secondary : bg #F5F5F5 / text #1A1A1A / radius 12px
Danger : bg #FF6B6B / text #FFFFFF / radius 12px
크기 : height 최소 48px (모바일 터치 영역 확보)

### 툴팁 (금융 용어 설명)

- 어려운 금융 용어 첫 등장 시 말풍선 툴팁 필수 제공
- 툴팁 내용은 초등학생도 이해할 수 있는 수준으로 작성
- 툴팁 최대 2줄 이내
  예) "전세보증금" 탭 시
  → "집주인에게 맡기는 큰 보증금이에요.
  계약이 끝나면 돌려받을 수 있어요."

## 페이지별 레이아웃 원칙

### 홈 화면

섹션 구성 순서 (고정):

1. 오늘의 한 줄 금융 상식 카드
2. 상황별 맞춤 탭 (새내기 / 취준생 / 신입사원)
3. 오늘의 퀴즈 CTA
4. 카테고리 탐색 (🏠 부동산 / 📋 세금 / 💰 재테크)
5. 인기 콘텐츠 / 신규 콘텐츠

### 콘텐츠 상세 페이지

섹션 구성 순서 (고정):

1. 30초 요약 카드 (핵심 3줄)
2. 쉬운 말 풀이 본문
3. 나에게 필요한 이유
4. 핵심 체크리스트
5. 관련 퀴즈 바로 풀기 CTA

### 퀴즈 페이지

진행 순서 (고정):

1. 배경 지식 30초 카드
2. 문제 + 선택지
3. 정답 해설 (쉬운 말)
4. 오늘 획득한 지식 요약

### 공통 레이아웃

- 모바일 최대 너비 : 390px
- 데스크탑 최대 너비 : 1200px
- 페이지 좌우 패딩 : 모바일 20px / 데스크탑 40px
- 섹션 간 간격 : 32px

## UX 원칙

### 로딩 상태

- 데이터 로딩 중에는 반드시 스켈레톤 UI를 표시한다
- 스피너 단독 사용 금지 (레이아웃 깨짐 방지)
- 로딩 시간 3초 초과 시 안내 메시지 표시

### 빈 상태 (Empty State)

- SSAC 마스코트와 함께 친근한 안내 메시지 표시
  예) 퀴즈 기록 없음
  → 🌱 "아직 푼 퀴즈가 없어요. 첫 퀴즈에 도전해볼까요?"

### 에러 상태

- 기술적 오류 메시지 노출 금지
- 사용자가 다음에 할 수 있는 행동을 함께 안내
  예) 네트워크 오류
  ❌ "NetworkError: Failed to fetch"
  ✅ "연결이 불안정해요. 잠시 후 다시 시도해주세요."

### 인터랙션

- 버튼 탭 시 즉각적인 피드백 (opacity 또는 scale 변화)
- 페이지 전환 시 fade-in 트랜지션 (300ms ease)
- 중요한 액션(탈퇴, 삭제 등)은 반드시 확인 다이얼로그 제공

### 접근성

- 모든 이미지에 alt 텍스트 필수
- 터치 영역 최소 48x48px 확보
- 키보드 탐색 가능하도록 focus 스타일 명시
- prefers-reduced-motion 미디어 쿼리 대응

### 콘텐츠 난이도 표시 원칙

- 모든 콘텐츠에 난이도 태그 필수 표시
- 난이도는 🌱 씨앗 / 🌿 새싹 / 🌳 나무 3단계로만 구분
- 예상 소요 시간 함께 표시 (예: "3분이면 충분해요")

## 에이전트 준수 규칙

### 디자인 작업 전 체크리스트

□ 컬러 토큰을 사용하는가 (임의 색상 사용 금지)
□ 폰트 스케일을 준수하는가
□ 카드 컴포넌트 구조를 따르는가
□ 금융 용어에 툴팁이 적용되었는가
□ 로딩 / 빈 상태 / 에러 상태가 모두 구현되었는가
□ 모바일 터치 영역 48px 이상인가
□ WCAG AA 명도 대비를 충족하는가

### 금지 패턴

- 컬러 토큰 외 임의 색상 하드코딩 금지
- 금융 전문 용어 툴팁 없이 단독 사용 금지
- 빈 상태에서 빈 화면 그대로 노출 금지
- 기술적 에러 메시지 사용자에게 직접 노출 금지
- 터치 영역 48px 미만 버튼 생성 금지

### DESIGN.md 업데이트 규칙

- 새로운 컴포넌트 추가 시 DESIGN.md에 반드시 명세 추가
- 컬러 / 타이포 변경 시 DESIGN.md 먼저 수정 후 코드 반영
- DESIGN.md에 없는 디자인 결정은 사용자에게 확인 후 진행
