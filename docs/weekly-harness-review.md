# 주간 하네스 리뷰 체크리스트

> 매주 스프린트 리뷰 또는 금요일 종료 전 팀이 함께 검토합니다.
> 이 파일 자체는 편집하지 않습니다. 각 주차별 기록은 아래 템플릿을 복사해 작성하세요.

---

## 리뷰 목적

> 하네스는 "에이전트가 실수할 때마다 강화"하는 살아있는 시스템입니다.
> 이 리뷰가 없으면 같은 실수가 반복되어도 아무것도 바뀌지 않습니다.

---

## 주차별 리뷰 템플릿

```markdown
### 주간 리뷰 — YYYY-MM-DD

**참석자**: 

#### 1. 이번 주 에이전트 실수 패턴

| 실수 유형 | 발생 횟수 | PR 링크 |
|---|---|---|
| (예: app/에서 services 직접 import) | 2회 | #12, #15 |

#### 2. 반복 실수 → 하네스 추가 결정

| 실수 | 추가할 하네스 규칙 | 담당자 | 완료 기한 |
|---|---|---|---|
| | | | |

#### 3. 현재 docs/ 상태 점검

- [ ] `docs/architecture.md`가 현재 코드 구조를 반영하고 있는가?
- [ ] `docs/conventions.md`에 새로 합의된 규칙이 반영됐는가?
- [ ] `docs/quality.md`의 기술 부채 목록이 최신 상태인가?
- [ ] 이번 주에 해소된 기술 부채가 있다면 등급 업데이트했는가?
- [ ] `docs/decisions/`에 새 ADR이 필요한 결정이 있었는가?

#### 4. 다음 주 하네스 우선순위

1. 
2. 
3. 

#### 5. 팀 메모

(자유 기록)
```

---

## 누적 실수 패턴 트래커

> 여러 주에 걸쳐 반복되는 실수를 여기에 기록합니다.
> 3회 이상 반복되면 ESLint 규칙 또는 AGENTS.md 금지 목록에 반드시 추가하세요.

| 실수 유형 | 총 발생 횟수 | 하네스 추가 여부 | 관련 파일 |
|---|---|---|---|
| (예: any 타입 사용) | 0 | ✅ ESLint Rule 1 | `eslint.config.mjs` |
| (예: app→services 직접 import) | 0 | ✅ ESLint Rule 4 | `eslint.config.mjs` |

---

## 하네스 강도 목표

| 시점 | 목표 |
|---|---|
| 현재 (초기) | 빌드 + 린트 + 포맷 + 타입 체크 자동화 |
| 1개월 후 | 테스트 커버리지 최소 기준 추가 |
| 3개월 후 | 레이어 의존성 완전 자동 검사 (dependency-cruiser) |
